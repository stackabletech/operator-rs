Stackable Data Platform — operator IT: fixes & failure analysis

For operator devs · window 2026-02-20 → 2026-06-28 · 100 red/unstable runs across 15 operators · Suggested fixes · Failure categories

Intention / how to use this

This is the result of a preliminary investigation and is, for now, mostly informational. Nothing here is a committed plan. Everyone is welcome to dig around — expand the fixes and categories, follow the log links, and see what's here if you're interested. New weekly test runs will be added to the lists as they come in. If and when we actually start fixing this, the suggested fixes will be transferred into GitHub issues for tracking; until then treat them as leads, not tickets.

What this is

Every failed/unstable weekly IT run classified into root-cause categories, representative runs deep-dived via the Jenkins test-output.log and (where still available) the T2 OpenSearch logs, distilled into the ranked fixes below. Start with the fixes. The failure-category drill-down follows; the two are cross-linked (each fix lists the categories it covers, each category lists the fixes that address it).

Limitations — read before acting

  • OpenSearch logs age out after ~2 weeks. Of 95 failed runs we tried to pull, only 10 still had logs; the rest are gone — so most root causes older than ~2026-06-18 are inferred from the kuttl test-output.log only.
  • Known vs conjecture. known affected = the run demonstrably failed in this area (kuttl --- FAIL and/or live-log confirmation). conjecture = attributed by pattern; mechanism not verified for that run (usually logs aged out). 🔬 verified = confirmed from live logs.
  • Teardown mechanism proven on 2 runs only (hbase#117, hive#114); the other ~30 teardown runs share the symptom but their exact blocker is conjecture.
  • Fix↔category is many-to-many (multi-cause runs); counts next to each link show shared runs.
  • 📊 OpenSearch links open the T2 Dashboards filtered to that run's cluster+time window (SSO login needed); targeted ones jump to the exact section. 📄 = Jenkins test-output.log artifact (always available).
